# Suggestions

This document captures product suggestions and guiding ideas for `notes.md`.

## Product Suggestions

### Add a "Today" or "Home" View Early

The app should have a clear entry point instead of dropping the user into a raw file list.

A "Today" or "Home" view can help the product feel more like a notes app and less like a file browser. It can later become the place where the user sees recent notes, daily notes, shortcuts, or quick actions.

### Support Quick Open and Quick Capture as First-Class Actions

These workflows fit the terminal very well and should be treated as core product actions.

Quick open makes it easy to jump to an existing note without navigating through folders. Quick capture makes it easy to save an idea immediately without breaking flow.

### Prefer Opinionated Defaults Early

The first versions of the product should avoid too much configuration.

Strong defaults are better than adding many settings too early. The app should feel usable out of the box before investing in advanced customization such as themes, custom keybindings, or complex sync behavior.
